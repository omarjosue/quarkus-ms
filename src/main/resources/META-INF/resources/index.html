Conekta

Quarkus Microservice

Architecture

  • REST resources: src/main/java/mx/conekta/qk/api.
  • Service stereotypes: src/main/mx/conekta/qk/service.
  • Utility classes: src/main/mx/conekta/qk/util.
  • Static assets: src/main/resources/META-INF/resources.
  • Application configuration: src/main/resources/application.properties.

Deployment

Run the application in dev mode using: mvn compile quarkus:dev.

Package it for production using: ./mvnw package. It will produce quarkus-ms-1.0.0-SNAPSHOT-runner.jar in the /target directory.

Run a native executable with: ./target/quarkus-ms-1.0.0-SNAPSHOT-runner using: ./mvnw package -Pnative or build in a container using: ./mvnw package -Pnative -Dquarkus.native.container-build=true.

Installed Features

amazon-dynamodb, amazon-s3, cdi, kubernetes, kubernetes-client, rest-client, rest-client-jackson, resteasy, smallrye-health, smallrye-metrics, smallrye-openapi, swagger-ui among others defined in pom.xml.
